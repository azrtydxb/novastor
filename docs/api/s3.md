# S3 API Compatibility

NovaStor provides an S3-compatible HTTP API through the S3 Gateway (`novastor-s3gw`). This page documents supported operations, authentication, and known limitations.

## Endpoint

The S3 Gateway listens on HTTP by default (port 9000). Access it using path-style URLs:

```
http://<s3-gateway-service>:9000/<bucket>/<key>
```

Virtual-hosted-style URLs (`<bucket>.s3.example.com`) are not supported.

## Authentication

### AWS Signature Version 4 (SigV4)

The gateway validates the access key from `AWS4-HMAC-SHA256` Authorization headers:

```
Authorization: AWS4-HMAC-SHA256
  Credential=AKID/20250115/us-east-1/s3/aws4_request,
  SignedHeaders=host;x-amz-date,
  Signature=<hex-signature>
```

The gateway extracts the access key from the `Credential` field and verifies it against the configured `--access-key`. Full signature verification (HMAC computation) is performed for presigned URLs.

### Legacy V2 Authentication

Also supported for backward compatibility:

```
Authorization: AWS <access-key>:<signature>
```

### Presigned URLs

The gateway supports presigned GET URLs with full signature verification:

```
http://<host>/<bucket>/<key>
  ?X-Amz-Algorithm=AWS4-HMAC-SHA256
  &X-Amz-Credential=<access-key>/<date>/us-east-1/s3/aws4_request
  &X-Amz-Date=<iso8601>
  &X-Amz-Expires=<seconds>
  &X-Amz-SignedHeaders=host
  &X-Amz-Signature=<hex-signature>
```

Presigned URL signature verification:

1. Validates the `X-Amz-Algorithm` is `AWS4-HMAC-SHA256`
2. Checks `X-Amz-Date` and `X-Amz-Expires` for URL expiration
3. Recomputes the signature using the server's secret key
4. Compares the computed signature with the provided `X-Amz-Signature` using constant-time comparison (`hmac.Equal`)

### Configuration

Set credentials via the S3 gateway flags or Helm values:

```bash
# Command line
novastor-s3gw --access-key=AKID --secret-key=SECRET

# Helm values
s3gw:
  accessKey: "AKID"
  secretKey: "SECRET"
```

!!! warning
    If `--access-key` and `--secret-key` are not set, the gateway rejects all requests with `AccessDenied`.

## Usage Examples

### AWS CLI

```bash
# Configure the CLI
export AWS_ACCESS_KEY_ID=AKID
export AWS_SECRET_ACCESS_KEY=SECRET
export AWS_DEFAULT_REGION=us-east-1

# Create a bucket
aws s3 mb s3://my-bucket \
  --endpoint-url http://novastor-s3gw:9000

# Upload a file
aws s3 cp myfile.txt s3://my-bucket/path/to/myfile.txt \
  --endpoint-url http://novastor-s3gw:9000

# Download a file
aws s3 cp s3://my-bucket/path/to/myfile.txt downloaded.txt \
  --endpoint-url http://novastor-s3gw:9000

# List objects
aws s3 ls s3://my-bucket/ \
  --endpoint-url http://novastor-s3gw:9000

# Delete an object
aws s3 rm s3://my-bucket/path/to/myfile.txt \
  --endpoint-url http://novastor-s3gw:9000
```

### Python (boto3)

```python
import boto3

s3 = boto3.client(
    "s3",
    endpoint_url="http://novastor-s3gw:9000",
    aws_access_key_id="AKID",
    aws_secret_access_key="SECRET",
    region_name="us-east-1",
)

# Create bucket
s3.create_bucket(Bucket="my-bucket")

# Put object
s3.put_object(
    Bucket="my-bucket",
    Key="hello.txt",
    Body=b"Hello, NovaStor!",
    ContentType="text/plain",
)

# Get object
response = s3.get_object(Bucket="my-bucket", Key="hello.txt")
data = response["Body"].read()

# Multipart upload
mpu = s3.create_multipart_upload(Bucket="my-bucket", Key="large-file.bin")
upload_id = mpu["UploadId"]

part1 = s3.upload_part(
    Bucket="my-bucket",
    Key="large-file.bin",
    UploadId=upload_id,
    PartNumber=1,
    Body=b"x" * (5 * 1024 * 1024),  # 5 MiB minimum part size
)

s3.complete_multipart_upload(
    Bucket="my-bucket",
    Key="large-file.bin",
    UploadId=upload_id,
    MultipartUpload={
        "Parts": [{"PartNumber": 1, "ETag": part1["ETag"]}]
    },
)

# Generate presigned URL
url = s3.generate_presigned_url(
    "get_object",
    Params={"Bucket": "my-bucket", "Key": "hello.txt"},
    ExpiresIn=3600,  # 1 hour
)
```

### Go

```go
package main

import (
    "context"
    "fmt"
    "strings"

    "github.com/aws/aws-sdk-go-v2/aws"
    "github.com/aws/aws-sdk-go-v2/config"
    "github.com/aws/aws-sdk-go-v2/credentials"
    "github.com/aws/aws-sdk-go-v2/service/s3"
)

func main() {
    cfg, _ := config.LoadDefaultConfig(context.TODO(),
        config.WithCredentialsProvider(
            credentials.NewStaticCredentialsProvider("AKID", "SECRET", ""),
        ),
        config.WithRegion("us-east-1"),
    )

    client := s3.NewFromConfig(cfg, func(o *s3.Options) {
        o.BaseEndpoint = aws.String("http://novastor-s3gw:9000")
        o.UsePathStyle = true
    })

    // Put object
    _, err := client.PutObject(context.TODO(), &s3.PutObjectInput{
        Bucket: aws.String("my-bucket"),
        Key:    aws.String("test.txt"),
        Body:   strings.NewReader("Hello from Go"),
    })
    if err != nil {
        fmt.Printf("Error: %v\n", err)
    }
}
```

## Data Model

### Object Metadata

Each object stored in NovaStor carries the following metadata:

| Field | Description |
|---|---|
| `Bucket` | Bucket name |
| `Key` | Object key (full path) |
| `Size` | Object size in bytes |
| `ETag` | MD5 hash of the object content |
| `ContentType` | MIME type |
| `UserMeta` | User-defined metadata (`x-amz-meta-*` headers) |
| `ChunkIDs` | List of chunk IDs composing the object data |
| `VersionID` | Version identifier (when versioning is enabled) |
| `ModTime` | Last modification timestamp (Unix epoch) |

### Multipart Upload

| Field | Description |
|---|---|
| `UploadID` | Unique upload identifier |
| `Bucket` | Target bucket |
| `Key` | Target object key |
| `Parts` | Ordered list of uploaded parts |
| `CreationDate` | Upload initiation timestamp |

Each part contains:

| Field | Description |
|---|---|
| `PartNumber` | Part sequence number (1-10000) |
| `Size` | Part size in bytes |
| `ETag` | MD5 hash of the part content |
| `ChunkIDs` | Chunk IDs for this part's data |

## Limitations

| Feature | Status | Notes |
|---|---|---|
| Virtual-hosted-style URLs | Not supported | Use path-style only |
| Bucket ACLs | Not supported | Single-tenant credentials |
| IAM policies | Not supported | Single access key pair |
| Cross-region replication | Not supported | Single-cluster only |
| S3 Select | Not supported | -- |
| Object Lock | Not supported | -- |
| Lifecycle rule enforcement | Partial | Configuration supported, enforcement not implemented |
| Bucket notifications | Not supported | -- |
| Server-side encryption headers | Partial | Configuration supported, encryption not implemented |
| Transfer acceleration | Not supported | -- |
| Batch operations | Not supported | -- |
| ListObjectsV1 | Not supported | Use ListObjectsV2 |
| Multi-object delete | Not supported | Delete objects individually |

## Error Responses

S3 errors are returned as XML in standard AWS format:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Error>
  <Code>NoSuchBucket</Code>
  <Message>The specified bucket does not exist</Message>
</Error>
```

Common error codes:

| Code | HTTP Status | Description |
|---|---|---|
| `AccessDenied` | 403 | Authentication failed or credentials do not match |
| `NoSuchBucket` | 404 | Bucket does not exist |
| `NoSuchKey` | 404 | Object key does not exist |
| `BucketAlreadyOwnedByYou` | 409 | Bucket already exists |
| `BucketNotEmpty` | 409 | Cannot delete non-empty bucket |
| `InvalidRequest` | 400 | Malformed request |
| `MethodNotAllowed` | 405 | HTTP method not allowed for this resource |
| `SignatureDoesNotMatch` | 403 | Presigned URL signature verification failed |
